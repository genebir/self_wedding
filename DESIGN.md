# DESIGN.md — 맑음 디자인 시스템 (v2)

> 디자인의 단 하나의 직무: **"이 서비스는 나한테 팔 게 없구나"를 3초 안에 느끼게 하는 것.**
> 웨딩 업계의 시각 언어(금박, 필기체, 긴급 배너, 팝업)는 마진의 언어다. 우리는 그 반대편에 선다.

---

## 1. 디자인 철학

Apple의 절제된 디자인 언어를 차용하되, 웨딩의 온기를 입힌다.

- **여백이 신뢰다.** 팔 게 없으니 화면이 조용할 수 있다. 밀도 높은 배너·뱃지·팝업은 전부 금지.
- **숫자는 장식하지 않는다.** 가격·표본수·분포는 데이터 그 자체로 아름다워야 한다. 그래프에 그라데이션·3D·과장 스케일 금지.
- **즐거움은 마이크로 인터랙션으로.** 체크리스트 완료, 예산 목표 달성 같은 *사용자의 진짜 진전*의 순간에만 delight를 쓴다. 우리 좋으라고(가입, 결제) 쓰는 delight는 다크패턴이다.

## 2. 컬러 토큰

| 토큰 | 값 | 용도 |
|------|-----|------|
| `ivory-bg` | `#FBF8F4` | 기본 배경. 순백(#FFF)이 아니라 따뜻한 아이보리 |
| `blush` | `#F5D7DC` | 연분홍. 서피스 강조, 선택 상태 |
| `blush-deep` | `#D98A96` | 주요 액션, 활성 상태 |
| `ink` | `#2B2B33` | 본문. 순검정 금지 |
| `ink-soft` | `#6E6E78` | 보조 텍스트, 캡션 |
| `trust-green` | `#4A7C59` | 신뢰 등급 A, 인증 마크 전용 |
| `flag-amber` | `#C4863B` | 견적 디코더 지뢰 플래그 전용 |

**규칙:** `trust-green`과 `flag-amber`는 각각 신뢰 표시와 경고 표시 외에 절대 다른 용도로 쓰지 않는다. 색이 의미를 독점해야 사용자가 색만 보고 신뢰/주의를 판독할 수 있다.
빨강 계열 경고는 쓰지 않는다 — 우리 화면에 "긴급"은 존재하지 않는다.

## 3. 타이포그래피

| 역할 | 서체 | 비고 |
|------|------|------|
| 디스플레이/헤딩 | Pretendard (600–700) | 한글 우선. 자간 -1% |
| 본문 | Pretendard (400–500) | 행간 1.6 |
| 숫자/데이터 | 탭ular figures 필수 | 금액·표본수는 자릿수 정렬. `font-variant-numeric: tabular-nums` |

금액은 언제나 `1,240,000원`처럼 완전 표기 + 보조로 `124만원` 병기. 축약만 쓰면 스케일 감각이 무뎌진다.

## 4. 시그니처 컴포넌트

### 4.1 정직한 가격 카드 (Honest Price Card) — v1 계승
분포를 보여주는 유일한 방식. 구성 요소가 하나라도 빠지면 렌더링하지 않는다:
- 중앙값 (크게) + 사분위 범위 (수평 분포 바)
- **표본 문장:** "32명이 실제로 낸 금액 기준" — 이 문장이 카드의 헤드라인이다
- 신뢰 등급 구성 (A/B/C/D 비율 도트)
- 기간·지역 조건
- "이 숫자는 어떻게 만들어졌나" 링크
- **n<10 상태:** 카드 자체가 "아직 표본 부족" 상태로 렌더. 흐릿한 가짜 분포로 유혹하지 않는다.

### 4.2 스코프 체크 (Scope Sheet) — v2 신규
예산 항목·견적의 `scope`를 표현하는 표준 컴포넌트. 숨은비용 사전과 연동.
- 포함 ✓ / 불포함 ✗ / **미확인 ?** 3-state. "미확인"이 기본값 — 모르는 걸 포함으로 착각하게 두지 않는다.
- 미확인 항목에는 "업체에 물어볼 질문" 문구를 자동 제안 (예: "원본 파일 제공이 포함인가요?")

### 4.3 디코더 플래그 (Decoder Flag) — v2 신규
견적 디코더의 지뢰 표시. `flag-amber` 단색, 아이콘 하나, 한 줄 설명 + 사전 링크.
톤 규칙: 업체를 비난하지 않는다. "이 항목은 별도 청구되는 경우가 많아요"처럼 **사실과 대비만** 말한다.

### 4.4 진전 마커 (Progress Moments)
체크리스트 완료·예산 마일스톤에서만 등장하는 blush 계열의 절제된 축하 모션 (200ms 내, reduced-motion 존중).

## 5. 카피 원칙

- 우리가 아는 것("32명 기준 중앙값")과 모르는 것("표본 부족")을 항상 구분해서 쓴다.
- 과장 어휘 금지 리스트: 최저가, 보장, 단독, 마감임박, 특가, 무료(조건부일 때).
- 사용자를 부르는 말은 "예비부부"가 아니라 그냥 2인칭. 역할을 규정하지 않는다.
- 에러·빈 화면은 사과하지 않고 다음 행동을 안내한다. 빈 분포 화면 = 기여 초대장.

## 6. 접근성·품질 바닥선

- 대비: blush 계열은 배경 전용, 텍스트 대비 4.5:1 이상 유지
- 키보드 포커스 가시화, reduced-motion 존중
- 모바일 퍼스트 — 결혼 준비는 이동 중·상담 중에 이뤄진다. 견적 디코더는 카메라 업로드가 1차 동선
